Getting the LMM CLLI name

If there are no communication problems with the CM, the default CLLI should automatically be set during the LMM GUI startup. If the CLLI is not already set during initialization, refresh the page.

Note: If you have any problem, click the link Troubleshooting.

Reconnecting to LMM server

Use this procedure to reconnect to the LMM server. A lost connection is indicated by a Red LMM status button. If a server fault occurs during GUI startup (a case when the C20 Management Tools server is down), the CM CLLI will not be set automatically. The CLLI field at the header of the GUI will be blank.

Example of lost connection

The C20 Management Tools server must be running and the applications in ready status.

Action

Perform the following steps to complete this procedure.

At your workstation

1 Ensure the C20 Management Tools server is back up and running with the applications in ready status.

2 Refresh the page in order to establish a connection with the LMM server again.

Note: If the server fault occurred during GUI startup, performing this step will automatically connect to the default CLLI. Once connected, the default CLLI will show up on the header of the GUI.

3 Check the status of the LMM Server box, which is green when the connection is re-established, as shown below.

4 You have completed this procedure.

Note: If you have any problem, click the link Troubleshooting.

Canceling pending CPD requests with LMM

Use this procedure to cancel pending CPD requests. This option is effective only when the Auto Termination timer runs out. Auto Termination must be enabled.

Action

Perform the following steps to complete this procedure.

At your workstation

1 On the Preferences menu, enable Auto Termination then Cancel pending CPD requests.

2 You have completed this procedure.

 Setting the LMM auto refresh rate

Use this procedure to set the auto refresh rate.

Action

Perform the following steps to complete this procedure.

At your workstation

1 On the Preferences menu, enable Auto Refresh.

2 Select the new value and click Save.

Note: The minimum is 5 seconds, and the maximum is 60 seconds.

3 You have completed this procedure.

 Disabling the LMM auto refresh

Use this procedure to disable the auto refresh.

Action

Perform the following steps to complete this procedure.

At your workstation

1 On the Preferences menu, disable Auto Refresh then Auto-Refresh (value) to disable automatic refresh of the display.

2 Verify that automatic refresh is disabled by viewing the Status window and the warning messages.

3 You have completed this procedure.

 Setting the LMM auto termination value

An auto termination timer is started when there is no activity on the LMM GUI. When the timer expires, lines are no longer refreshed and pending CPD requests are cancelled. Use this procedure to set the auto termination value.

Action

Perform the following steps to complete this procedure.

At your workstation

1 On the Preferences menu, enable Auto Termination

2 Select a new value for the timeout and click OK.

Note: The minimum is 1 minutes, and the maximum is 480 minutes.

3 You have completed this procedure.

 Controlling the number of lines displayed by the LMM GUI

The LMM allows you to control the number of lines that appear in the LMM GUI. You can choose the number of lines from the dropdown list. With auto-refresh enabled, only the lines posted and visible at the GUI will be refreshed As the user navigates using the “Next” and “Prev” buttons, the corresponding posted visible set will be refreshed. Use this procedure to modify the number of displayed lines in the LMM GUI.

Action

Perform the following steps to complete this procedure.

At your workstation

1 On the Home page, after posting a DN or gateway, then select the number of lines to display from the Rows below the table.

Selecting the number of lines to display

2 If the number of displayed lines on GUI is larger than the value you chose in the previous step, the “Next” and “Prev” buttons will be automatically enabled, if they are not already enabled. Use the “Next” and “Prev” buttons to verify that the new display settings become effective.

3 If the number of displayed lines on GUI is less than the value you chose in the previous step, the “Next” and “Prev” buttons will be enabled when the set value is reached.

4 You have completed this procedure.
