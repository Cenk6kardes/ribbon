Functional Description

The audit panel allows for the configuration of audits, viewing of reports, and execution of audit runs.


Start

  1. At the C20 Management Tools GUI, click Audit > C20 Data Integrity
  2. Select the C20 component that you wish to audit.
    Select the C20 Call Server Data Integrity Audit check box — if you wish to verify that the data held by the GWC Manager database and the call server match.
    Select both C20 Call Server Data Integrity Audit and AS data integtrity audit boxes — if you wish to audit both components.
  3. In the Schedule pane, select Now to run the audit immediately or select the day and time to schedule the audit.
  4. Click Run to start the audit or Save for the scheduled audit.
    If you wish to cancel the process, click the Close button
    During a C20 audit, the system displays a progress window with a progress bar.

    Schedule For Later:


  5. The audit may take a few minutes to complete. When the audit ends successfully, click View Report to view the audit results.
  6. Click the Close button to close the Audit Status window.
  7. To view the audit report, follow procedure View C20 Data Integrity Audit Reports.

Start

  1. At the C20 Management Tools GUI, click Audit > Line Data Integrity
  2. Read the following guidelines, then select the elements that you wish to audit.
    If you select the Integrity Audit option (to select all GWCs and associated gateways or all logical groups), the system confirms whether to audit the Call Agent DN during the Line Integrity Audit.
    Click Yes if you wish to continue. Otherwise, click No.
  3. In the New Audit pane, select the configuration for the audit.
    The Granular Line Audit list on the left displays all the small and large line GWCs and the associated gateways, as well as the logical line groups (LGRP). You can display or hide any of the sub-lists by expanding or collapsing the lists.
    If you wish to select more than one item at the same time, click each item that you want to select.
    Do not select GWCs and LGRPs for the same audit. If you attempt to select both, the system displays an error message.
  4. Click Add to add the configuration selections from the Granular Line Audit to the Selected list.
  5. If necessary, repeat the selection process until all the items that you want to audit appear in the Selected window.
    If you need to remove an item from the Selected table, click the item to highlight it, then click < to remove the selection.
  6. Click Run to start the audit or Save for the scheduled audit.
    During a line audit, the system displays a progress window, which includes the progress bar and the current operation.

This procedure describes how to perform an on-demand data integrity audit of V5.2 interfaces for a C20 in the international market.
Use this procedure to check for defective data after you have done V5.2 interface provisioning, or if you suspect there is a problem with V5.2 provisioning.
When the audit is running, suitable locks are in place that disable provisioning operations.

Start

  1. At the C20 Management Tools GUI, click Audit > V5.2 Data Integrity
  2. In the Schedule pane, select Now to run the audit immediately or set the day and time for a scheduled audit.
  3. Click Run to start the audit or click Save to schedule the audit.
    During the audit, the system displays the Audit Status message box.
  4. The audit may take a few minutes to complete. When the audit completes successfully, the system displays one of two types of messages:
    V5.2 Data Integrity Audit Completed. No Problems Found.
    V5.2 Data Integrity Audit Completed. 2 Problems Found.
    If the audit fails, the system displays the V5.2 Data Integrity Audit Failed to Complete message with an error message indicating the reason.

    5. Run Now:


    Schedule For Later:


  5. Click the Close button to close the Audit Status window.
  6. To view the V5.2 audit report, follow procedure View V5.2 Data Integrity Audit Reports.

Use this procedure to view and, if required, abort currently running audits. You can view and, if necessary, abort any of audit.

Start

  1. At the C20 Management Tools GUI, click Audit > Current Audit.
  2. If you wish to abort one of the running audits, click the appropriate row to select the audit. Your selection is highlighted.
    Click the Abort button.
    At the confirmation window, click OK to continue.
    The audit originator receives an error message indicating that the audit was aborted and identifying the user that aborted the audit.
