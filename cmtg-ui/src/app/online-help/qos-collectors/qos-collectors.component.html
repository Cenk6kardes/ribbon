Functional Description

The QoS Collectors panel provides access to configuration data for QoS collectors associated with the selected gateway controller.

Displays the list of QoS collectors and allows the user to associate one with the Gateway Controller. Refer to the Associate a QoS Collector procedure for more information.

Use this procedure to associate a QoS Collector to a Gateway Controller.

  1. Display the Provisioning panel.
  2. Select the Qos Collectors panel.
  3. Select the Associate...button. A dialog will be displayed
  4. Click on the QoS Collector you would like to associate to the Gateway controller. Click outside the popup box to cancel.
  5. The QoS Collector will appear in the list following successful association.

Disassociates the selected QoS collector from the Gateway Controller. Refer to the Disassociate a QoS Collector procedure for more information.

The following procedure is used to disassociate a Media Proxy from a Gateway Controller.

  1. Display the Provisioning panel.
  2. Select the Qos Collectors panel.
  3. Select the QoS Collector you would like to disassociate from the Gateway Controller.
  4. Select the Disassociate button.
  5. Click the Yes button to confirm the request, or the No button to abort the request.
  6. The QoS Collector will be removed from the list following successful disassociation.

Set QoS collection on the Gateway Controller. Refer to the Set QoS Collection procedure for more information.

Use this procedure to set QoS Collection status for a Gateway Controller.

  1. Display the Provisioning panel.
  2. Select the Qos Collectors panel.
  3. Select the check boxes on QoS Collectors panel to configure QoS Collection, and then click Apply button to set QoS Collection.
  4. Move cursor near the check boxes and the first column in RTCP-XR blocks table to get detailed reporting metrics information.

Refresh this table to get the lastest records about Qos Collectors(maybe changed by other users).
