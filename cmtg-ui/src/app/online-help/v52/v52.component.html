Functional Description

The V5.2 panel provides access to configuration of V5.2 trunk services on a Gateway Controller (GWC) node
Procedures in this section apply to international markets only.


V5.2 Interface Browser Attributes

Use this procedure when you are adding V5.2 interfaces and associating physical E1 links to a specific interface.

Use this procedure to perform the following tasks:

  1. Add and datafill a V5.2 Interface.
  2. Map a bundle of up to 16 E1 carriers to the Interface.
  3. Assign a ring plan, provisioning profile and signaling profile

When adding an interface, the following guidelines apply:

  1. One interface cannot be spread over more than one GWC.
  2. One interface can be spread over one or more gateways.
  3. Links of several interfaces can be defined on one gateway

Start

  1. At the C20 Management Tools GUI, click V5.2.
  2. Click the New Interface button.
  3. Type the interface attributes for each of the fields as needed. Refer to V5.2 Interface Attributes.
  4. Click Add button.

Prerequisites

  1. Delete lines associated with the interface.
  2. De-provision all lines referenced on the interface.
  3. Deactivate the interface in the Call Agent using the maintenance level of the MAP.

Start

  1. At the C20 Management Tools GUI, click V5.2.
  2. From the Select a V5.2 Interface drop-down list, select the interface that you want to delete.
  3. Click the Delete Interface button.
  4. At the confirmation dialogue box, click OK

Use this procedure to modify the attributes of an existing V5.2 interface.

You cannot modify the following interface options:

  1. Maxlines (you cannot change the Access Node size of an existing interface).
  2. GWC-ID.

To make those changes, you must provision a new interface.

Prerequisites

You must deactivate the interface on the Call Agent using the maintenance level of the MAP before making the following modifications

  1. Delete lines associated with the interface.
  2. De-provision all lines referenced on the interface.
  3. Deactivate the interface in the Call Agent using the maintenance level of the MAP.

Before changing or deleting a link's mapping, you must busy the link on the Call Agent using the maintenance level of the MAP.


Start

  1. At the C20 Management Tools GUI, click V5.2.
  2. From the Select a V5.2 Interface drop-down list, select the interface that you want to modify.
  3. Modify the field values for the selected interface. Use V5.2 Interface Attributes to assist you when changing attribute values.
  4. Click the Save button when you are finished.
  5. At the confirmation dialogue box, click OK

Use this procedure to view provisioning details about the V5.2 interface.

Start

  1. At the C20 Management Tools GUI, click V5.2.
  2. From the Select a V5.2 Interface drop-down list, select the interface you wish to review.
  3. Review the field values for the V5.2 interface. For a description of the various fields, see V5.2 Interface Attributes.

V5PROV Attributes

Use this procedure when creating a new V5.2 provisioning template.

Use this procedure to create and datafill V5PROV table information relating to:

  1. Which channels and on which links are the signaling channels.
  2. What type of signaling is sent over which link.
  3. Which channels are used for backup signaling

Table V5PROV is referenced by the main CM table GPPTRNSL.

Prerequisites

  1. You must know the V5 variant ID that is provisioned on the access node. This needs to match the V5PROV template as datafilled for the Interface (in table GPPTRNSL). Consult your site system administrator for details on how to acquire this information.
  2. The c-channel information in this profile must match the provisioning profile in the access node.
  3. Tables V5RING and V5SIG must also be datafilled before the interface is fully provisioned.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5PROV.
  2. From the Identifier drop-down menu, select a provisioning template to modify by selecting its Identifier.
  3. Use V5PROV Attributes to add values to the field entries.
  4. Click Add button.

Use this procedure to remove a configured V5 provisioning template, when the template is no longer needed, or if it is being replaced by another template.

Prerequisites

  1. Check all existing V5.2 interfaces and modify them to ensure they are not referencing the template being removed. See V5.2 Interface Browser > View V5.2 Interface Browser

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5PROV.
  2. Click the Delete button.
  3. From the Delete Identifier drop-down list, select the template that you want to delete.
  4. Click Delete.

Use this procedure to make changes to the V5 interface provisioning template configuration.

Prerequisites

  1. You cannot modify a provisioning template that is currently being used by a V5.2 interface. If necessary, check all existing V5.2 interfaces and modify them to ensure they are not referencing the template being modified. See procedure View V5.2 Interface Browser > View V5.2 Interface Browser to perform this task.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5PROV.
  2. Click the Identifier drop-down menu and select a provisioning template to modify by choosing an identifier.
  3. Use V5PROV Attributes to review and modify the field entries
  4. Click Save.

Use this procedure to view the details about a configured V5 interface provisioning template.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5PROV.
  2. Select a provisioning template to review by selecting its Identifier.
  3. See V5PROV Attributes to determine the meaning and value of the field entries.

V5RING Attributes

Use this procedure to create V5 ring templates for the V5RING table which contain information relating to mappings between ringing cadences and ringing types. Table V5RING is referenced by the main CM table GPPTRNSL.

Use this procedure when a new V5.2 interface is being provisioned.

A default profile, identified as DEFAULT, is always provided in the template.

Prerequisites

  1. Tables V5PROV and V5SIG must also be datafilled before the interface is fully provisioned.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5RING.
  2. Set up a new template by adding an Identifier. Click the New V5Ring button and fill in the appropriate fields.
  3. Use V5RING Attributes to enter all of the ring attributes for a V5.2 ring template.
  4. Click Add button.

Use this procedure to delete an existing V5 ring template.

Prerequisites

  1. Ensure that the ring template to be removed is not currently used by a V5.2 interface.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5Ring.
  2. Click the Delete button.
  3. From the drop-down list, select the ring mapping identifier for the template that you want to delete. You cannot delete the default identifier.
  4. Click Delete.

Use this procedure to modify V5 ring templates for the V5RING table which contains information relating to mappings between ringing cadences and ringing types. Table V5RING is referenced by the main Call Agent table GPPTRNSL.

Prerequisites

  1. No V5.2 interface can be referencing this template while you are making changes. Check all existing interfaces and make the necessary changes to ensure they are not referencing the template you intend to modify. See V5.2 Interface Browser > View V5.2 Interface Browser.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5RING.
  2. Modify the required ring attributes for the selected V5.2 ring template. Use V5PROV Attributes.
  3. Click Save when you are finished modifying the ring template.

Use this procedure to view details about an existing V5 ring template.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5RING.
  2. Click the Identifier drop-down list and select a ring template identifier to review.
  3. Use V5PROV Attributes to review the fields, terms and the descriptions.

V5SIG Attributes

Use this procedure to create and datafill V5SIG table information relating to signaling characteristics. Table V5SIG allows a set of signaling characteristics to be defined as a signaling profile. There is a default profile provisioned on any switch. These characteristics include (but are not limited to) line attenuation, End-Of-Call signaling support and suppression indication. Table V5SIG is referenced by main CM table GPPTRNSL.

Use this procedure when provisioning a new V5.2 interface.

For V5.2, a line attenuation value of V5_DIGITAL is not currently supported.

Prerequisites

  1. Tables V5PROV and V5RING must also be datafilled before the interface is fully provisioned.
  2. This procedure assumes that you are already logged into the C20 Management Tools GUI and that you know the V5 Interface ID and the GWC node ID. Consult your site system administrator to acquire this information.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5SIG.
  2. Configure all of the signaling attributes for the desired V5.2 interface.
  3. See V5SIG Attributes for the attributes
  4. To add a new Identifier click the New V5SIG button and configure the appropriate fields.
  5. Click Add button.

Use this procedure to delete a configured interface signaling template.

Prerequisites

  1. Ensure that the signaling template to be removed is not currently being used by a V5.2 interface. V5.2 Interface Browser > View V5.2 Interface Browser

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5SIG.
  2. Click the Delete button.
  3. In the Delete Identifier dialogue box, click the drop-down list and select the identifier for the template that you want to delete.
  4. Click Delete.

Use this procedure to modify an existing interface signaling template.

Prerequisites

  1. Ensure that the signaling template to be removed is not in use by a V5.2 interface. See V5.2 Interface Browser > View V5.2 Interface Browser.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5SIG.
  2. Click the Identifier list and select the identifier for the template that you want to modify.
  3. Change the required attributes using V5SIG Attributes.
  4. Click Save when you are finished modifying the sig template.

Use this procedure to view the V5SIG table signaling attributes.

Start

  1. At the C20 Management Tools GUI, click V5.2 > V5SIG.
  2. Select the signaling template attributes to review using the Identifier list.
  3. For the field definitions, see V5SIG Attributes.

Use this procedure to view carrier-to-interface endpoint mapping information associated with a selected carrier or V5.2 interface.

Use this procedure when you need to determine the following information:

  1. The carriers associated with a specific V5.2 interface.
  2. The link IDs used to associate a carrier to an interface.
  3. The V5.2 interface terminating on a particular gateway.

Start

  1. At the C20 Management Tools GUI, click V5.2 Maintenance.
  2. In the Interface to Carrier tab, click the V5.2 Interface drop-down menu and select an interface
  3. Click Retrieve.
  4. Click the Carrier to Interface Mapping tab to view the V5.2 Interface ID-to-Carrier Name mapping.
  5. In the Gateway Name field, type the name of a gateway.
  6. Select the Wildcard check box. The wildcard option allows the system to examine all endpoints on a given gateway. With this option selected, the system displays any endpoints associated with a V5 interface.
  7. Click Get Mapping
