Functional Description

The Gateways panel provides access to configuration data for media gateways associated with the selected gateway controller.

A common search interface is located at the top, while the lower section contains a table to display the retrieved records and functions specific to Gateways.

Displays the Associate Media Gateway dialog to associate a Media Gateway to the selected Gateway Controller. Refer to Associate a Media Gateway procedure for more information.

Use this procedure to associate a Media Gateway to a Gateway Controller from the "C20 Management Tools" GUI. For those customers who are using an Operations Support System (OSS) interface to associate gateways via XML, please refer to the "OSSGate User Guide".
  1. There are two methods available to associate a Media Gateway.
    1. Select the Associate Media Gateway menu item from the Gateway Controllers page. The Associate Media Gateway dialog appears upon selection.
    2. An alternate method to associate a Media Gateway to a specific gateway controller is to: display the Provisioning panel for a specific gateway controller, select the Gateways panel, and then click the Associate... button. The Associate Media Gatewaydialog appears, with the Gateway controller name preconfigured.
  2. Enter the media gateway name in the Gateway name field, and IP address in the Gateway IP address field.
  3. If the gateway controller with which the media gateway is to be associated is known, select the appropriate gateway controller in the Gateway controller name field; otherwise if the selected profile is a small line this will be set as <auto select>. (This field is preconfigured and cannot be changed when associating the media gateway via the Gateways panel.)
  4. Select a profile for the media gateway from the Gateway profile name pull down menu.
    1. If the Gateway profile selected in Gateway profile name supports DQoS, the Gateway PEP Server radio option and ALG radio option will appear. Select the appropriate radio button and enter either the PEP server name or ALG name in the appropriate field.
    2. If the Gateway profile selected in Gateway profile name supports displaying physical location of the gateway, the LGRP Location display will appear. Enter the physical location data for this gateway in these fields. Note that the provisioning of the Frame and Unit fields in this display is restricted to certian type of gateways only, for example, to gateways whose endpoint type in the profile definition is set to "thirdPartyLens"
    3. Select the site from the Gateway site name pull down menu. This field is available only if the selected profile includes the LINE service type.
    4. Select a protocol type from the Protocol type pull down menu. Enter the protocol port in the Protocol port field and the protocol version in the Protocol version field.

If the association failed, clicking the Show Details button in the response dialog box displays the failure reason.

Removes the selected Gateway from the selected Gateway Controller. Refer to Disassociate a Media Gateway procedure for more information.

The following procedure is used to remove a Media Gateway from a selected Gateway Controller when using the "C20 Management Tools" GUI. For those customers who are using an Operations Support System (OSS) interface to disassociate gateways via XML, please refer to the "OSSGate User Guide".

Before disassociating a media gateway, all corresponding endpoints must be deleted. If an attempt is made to disassociate a media gateway with endpoints still provisioned against it, the disassociation will fail and a dialog box will be displayed with the failure reason.

  1. There are two methods available to disassociate a media gateway.
    1. Select the Disassociate Media Gateway menu item from the Configuration menu. The Disassociate Media Gateway dialog appears upon selection. Enter the media gateway name in the Gateway Name field.
    2. An alternate method to disassociate a media gateway is as follows. Display the Provisioning panel for a specific gateway controller, then select the GatewaysGateways panel. Next, display the desired gateway via the search interface provided in the Gateways panel. Select the gateway from the Gateway List and click the Delete button. The Disassociate Media Gateway dialog appears with the Gateway name field set to the selected gateway.
  2. A Confirm Gateway Deletion dialog will appear. Click Save to confirm the deletion or Cancel to cancel the deletion. Once the Save button is clicked a Please wait while processing… dialog will appear until a response dialog box appears. The response dialog box indicates whether the gateway deletion was successful or not. If the deletion failed, clicking the Show Details button in the response dialog box displays the failure reason.

Displays the Change Gateway dialog to modify Gateway configuration. Refer to Change Gateway Configuration procedure for more information.

Use this procedure to change the configuration of one or more Gateways.

  1. Display the Provisioning panel.
  2. Select the Gateways panel.
  3. Display the gateway(s) to be modified in the Gateway List by either specifying the name, or other criteria, in the Retrieval criteria field then clicking the Retrieve button, or by clicking the Retrieve All button.
  4. In order to change ALGs, PEP server, gateway capacity, profile, gateway protocol and IPAddress/port select the gateway(s) to be changed from the Gateway List and click the Change... button. This displays the Change Gateway dialog where the appropriate operation can be selected.
  5. Select a change operation from the drop down list.

    1. Change ALG

      This function changes the ALG association for the selected gateways. The ALG is selected from the drop down list.

    2. Change PEP server

      This function changes the PEP server association for the selected gateways. The PEP server is selected from the drop down list.

    3. Change gateway capacity

      This function changes the reserved capacity for the selected gateways.

    4. Change profile

      This function changes the profile for the selected gateways.

      Profile changing operation can also be done from the Bulk Action menu for more than one gateway.

    5. Change gateway IP Address (H.323 or SIPVOICE only)

      This function changes the IP Address and/or port of the selected gateway. Currently, this functionality is only supported for H.323 GWs that are associated with NATs or SIPVOICE GWs.

  6. Click the Save button to confirm the request or the Cancel button to abort the request. Following a confirmation, a dialog appears to display the results of the requested operation(s).
