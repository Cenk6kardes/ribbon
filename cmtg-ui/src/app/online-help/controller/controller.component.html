Functional Description

The Controller panel provides data specific to the selected Gateway Controller. The configuration data displayed may not be changed, with the exception of the profile. A description of the fields are provided below.

The General section displays the other miscellaneous information. Please refer to table below for a detailed description.

The Profile section displays the currently assigned profile and the list of capabilities associated with that profile. The profile may be changed by selecting the Change... button to the right of the current profile. Refer to the Change Gateway Controller Profile procedure for more information.

Change Gateway Controller Profile

Use this procedure to change the profile for a Gateway Controller.

  1. Display the Provisioning panel.
  2. Select the Controller panel.
  3. Go to the profile page of the controller Click the Select Profile dropdown menu.The GWC Profiles are displayed, or a message dialog indicating the current profile does not support changing. Select the new profile from the drop down list.

    4. Notice:

    Since CVM11, GWC whose profile is LINE_TRUNK_AUD_NA or LINE_TRUNK_AUD_INTL, can support flow through function. This means the GWC data of these two profile will be auto-added on Session Server, if flow through checkBox is selected when adding GWC or changing GWC profile to LINE_TRUNK_AUD_NA or LINE_TRUNK_AUD_INTL.

    Example of changing GWC profile and choosing flow through GWC data to Session Server:

    If flow throuth checkBox is selected while the GWC data had already been added on Session Server before, the operation of changing GWC profile will be failed for duplicate GWC data is requested to be added on Session Server. Once meeting this error scenario, un-selected the flow through checkBox when requesting to change GWC profile.

  4. Click the OK button to confirm the request, or the Cancel button to abort the request.
  5. Following a successful change, both units of the Gateway Controller must be reloaded to enable the change. An alarm is raised for each unit to indicate a data mismatch exists between the SESM server and the Gateway Controller. The alarm condition on the NDM Alarm Interface. In order to change the GWC Profile, the following operations must be executed on NDM:
    1. Lock and Unlock the inactive GWC unit.
    2. Warm swact the GWC.
    3. Lock and Unlock the new inactive GWC unit.

The Bearer Network and Codec Profile section displays the currently associated bearer network and codec profile. Please refer to table below for a detailed description of the fields. The codec profile may be changed by clicking the Change... button at the right of the current codec profile. Refer to the Change Gateway Controller Network Codec Profile procedure for more information.

Use this procedure to change the network codec profile for a Gateway Controller.

  1. Display the Provisioning panel.
  2. Select the Controller panel.
  3. Click on the Change icon at the right of the current codec profile. The Change GWC Codec Profile dialog is displayed. Select the new codec profile from the drop down list. Then click the ✓ button to apply this change, or ✖ to abort this operation.
  4. You will see the confirmation window.
  5. Following a successful change, settings of the new codec profile will be re-synchronized to both units of the Gateway Controller automatically. But if the synchronization fails, a warning message will be displayed. Perform a BSY/RTS of the unit to force re-synchronization of the codec information.
